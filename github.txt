[Conceptos claves de Git]

- Navegar entre las versiones gracias a los sanapshots. Esto has de 
cuenta que es una captura de el codigo, y cuando navegas entre estos
snapshots puedes modificar, copiar, cortar eliminar y cosas asi.

- Git es una herramienta que interactua directamente con el sistema 
operativo, asi que se maneja por la consola. Git te instala un tipo
de terminal, un bash. Es una consola con mejores funcionalidades.
tiene todos los conceptos de unix, sistema operativo linux o mac.

-En git hay 3 estados 1."Working Directory" 2."Staging Area" 
3."Repository"| Por lo que entendi. 1. Working Directory es el documento
que usaras localmente, el que solo se almacena en tu computador. 
2. Stating Area es cuando ya se sube a la nube pero no al repositorio
principal. 3. Repository, el repository es cuando ya se ha subido 
aal repositorio principal, o sea ya se guardo como una versiones.

- git checkout -- nombreDeArchivo.html : Te va a reescribir el codigo
que tenias cuando hiciste el ultimo snapshot

- git diff: Si por ejemplo editas un codigo opbviamente este va ser
diferente asi que pues aun asi lo puedes checar con git status, en
caso de que si sea diferente puedes usar git diff, lo que hace es 
que te da todo el codigo diferente de la version guardada en el 
snapshot al actual del working directory.

Si al ingresar git status sale un archivo con un slash "/" quiere
decir que es una carpeta.

si quieres ignorae un archivo o carpeta puedes crar un archivo tipo
.gitignore y aqui poner la ruta o nombre de los archivos a ignorar.

pudes agregar todo el contenido ignorado si agregas el archivo .gitignore
Cuando ignoras un archivo va a ser que cuando uses el git status no 
aparezca ese archivo mas que el .gitignore. 

Igual solo podras agregar esos archivos ignorados agregando el 
archivo .gitignore.
